## Summary

Исследованы **9 сегментных гипотез** для AI-ассистента с on-prem, методологией внедрения и полным набором функций (документы + email + встречи). **5 сегментов живы**, 2 убиты (Логистика, Тренинговые), 2 заморожены/отложены (Госорганизации, Девелоперы). ## Killed Segments

### Логистика / Транспорт — УБИТ (3 evidence-backed блокера)

**Блокер 1 — Ability to pay [E]:** При средней рентабельности транспортных компаний 4%, компания с выручкой 100M₽ зарабатывает ~4M₽ чистой прибыли. Продукт за 960K-1.32M₽/год = 24-33% всей чистой прибыли. Даже при выручке 500M₽+ (прибыль ~20M₽) продукт = 5-7% прибыли — слишком тяжёлая строка расходов для компании с тонкой маржой. Клиент в REDO-заказе сам отметил: "тут у них может не быть денег на цифровую трансформацию" — исследование подтвердило.

**Блокер 2 — ЭДО-триггер ложный [E]:** Обязательная электронная ТТН с сентября 2026 полностью обслуживается специализированными ЭДО-операторами (1С-ЭПД, Контур, Астрал, СБИС). Э-ТТН — юридически регулируемый документ с КЭП. Нулевое пересечение с функциональностью AI-ассистента для офисной работы. Триггер, который казался сильным в v1, оказался из другой категории продуктов.

**Блокер 3 — AI в логистике ≠ офисная рутина [E]:** По данным ассоциации "Цифровой транспорт и логистика", 45% компаний планируют AI — но use cases: оптимизация маршрутов, прогнозирование спроса, складская робототехника. Не офисная обработка документов и клиентская переписка. Конкретных примеров AI для офисной работы в средних логистических компаниях РФ не найдено.

### Тренинговые компании — УБИТ (2 evidence-backed блокера)

**Блокер 1 — Размер рынка [E]:** Тренинговых компаний с выручкой >50M₽ в РФ менее 500. Большинство — микро-предприятия и соло-тренеры, для которых продукт за 80-110K₽/мес несоразмерен масштабу бизнеса. Нет критической массы для FL-кампании.

**Блокер 2 — Ценовой fit [E]:** 80-110K₽/мес на 10-15 сотрудников = 6-8K₽/чел/мес — в 50 раз дороже iSpring LMS (131₽/чел/мес). Корпоративные университеты с бюджетами от 557M₽ существуют, но это enterprise-клиенты с другим циклом продаж (6-12 мес), что выходит за рамки FL-модели (1-3 недели). Пилоты клиента в "обучающих компаниях" вероятно относятся к корпоративным учебным центрам при крупных компаниях — не к средним тренинговым фирмам.

### Госорганизации — ЗАМОРОЖЕН (hard blocker: Реестр ПО)

Потенциал подтверждён: IT-бюджеты 3-15M₽/год, Нацпроект "Экономика данных" 457 млрд ₽, конкуренция слабая (CorpGPT — no-code, Directum Ario — другая категория). Малые электронные закупки до 5M₽ по 44-ФЗ позволяют обойти тендер. Однако: Постановление Правительства №1236 запрещает закупку ПО вне Реестра по 44-ФЗ, а CorpGPT уже в Реестре (запись №23046) → исключение "нет отечественного аналога" не работает. Регистрация занимает 2-4 мес. при 21% approval rate. **Рекомендация клиенту:** начать регистрацию немедленно, FL разморозит сегмент после получения записи в Реестре.

### Девелоперы / Строительство — ОТЛОЖЕН (рынок не готов)

Сигналы есть: ГК ФСК снизила время подготовки документов на 45%, ГК КОРТРОС создаёт "интеллектуальную базу знаний", 70% девелоперов заинтересованы в AI. Но строительство — одна из наименее цифровизованных отраслей РФ (IT-расходы <1% выручки). Конкурент НейроШтат уже предлагает 50 виртуальных AI-сотрудников с on-prem для строительных компаний. Generic продукт может не покрыть отраслевую специфику (СНиП, ГОСТ, методологии Минстроя). **Мониторить.** Не вкладывать ресурсы FL сейчас.

### Почему покупают

1. **Каждый новый разработчик первые 2 месяца работает вполсилы, потому что не может найти нужную документацию.** При зарплате 200-300K₽/мес вы теряете 200-300K₽ на каждом новом найме. Новые сотрудники тратят до 30% времени на поиск информации [MTS Web Services]. 43% компаний не имеют структурированного онбординга. → AI-ассистент делает нового сотрудника продуктивным с первого дня — он находит любую документацию за секунды.

2. **Когда уходит senior — вместе с ним уходят знания о вашем продукте. Восстановление стоит 2 недели всей команды.** Уход senior-разработчика = двухнедельный knowledge audit [MTS Web Services]. Знания раскиданы по чатам, PR, записям встреч — собрать их после увольнения почти невозможно. → AI непрерывно индексирует документы, переписки, записи встреч. Уход человека не обнуляет команду.

3. **Support эскалирует тикеты разработчикам, потому что не может найти ответ в Confluence. Каждая эскалация = 30 мин разработчика.** Инженеры тратят до 20% времени на поддержку документации [MTS Web Services]. При 10 эскалациях/день = 5 часов разработчиков ≈ 100K₽/мес. → AI ищет по всем источникам разом — включая записи митингов и почту. Support находит ответ сам.

4. **Ваши разработчики пользуются ChatGPT через VPN. Вы не контролируете какой код и данные утекают наружу.** ChatGPT функционально заблокирован в РФ (IP-блоки, оплата). Команды обходят через VPN на общем аккаунте. Внутренний код, переписки, данные клиентов — всё летит в OpenAI. → On-prem AI-ассистент: данные не покидают контур компании. Легальная замена ChatGPT для всей команды.

## Segment Sequence

```
IT-компании (P1) ──→ Производство (P2) ──→ Юр. фирмы (P2) ──→ Медицина (P3)
        │                                          │
        └──── Оптовая торговля (P1, параллельно)    └──→ Корп. юр. отделы (expansion)
```

**Почему такой порядок:**

1. **IT + Торговля параллельно (P1):** Максимально быстрый feedback loop. IT — самый быстрый цикл (2-6 недель), цифровые ЛПР, лучшая reference value. Торговля — крупнейший TAM, CRM-интеграторы как тёплый канал. FL запускает оба одновременно, данные за 2 недели определяют кто станет "основным".

2. **Производство (P2, со сдвигом 1-2 недели):** АСКОН-канал — стратегический актив. Нулевая конкуренция. Но осознание проблемы ниже → нужен другой messaging. FL начинает с малого объёма (20 звонков + 3 офиса АСКОН) параллельно с P1.

3. **Юр. фирмы (P2):** Малый TAM, но высокая боль и willingness to pay. Reference → корп. юр. отделы (10x expansion). Запуск после первых результатов P1, когда messaging отточен.

4. **Медицина (P3):** Только после кейсов из других сегментов. Консервативный рынок, конкуренты уже на месте, ЛПР недоступны — нужны "костыли" (reference, case studies) для входа.
