## Macro Signal from Research

Самый мощный сигнал — кросс-сегментный:

- **95% AI-пилотов в РФ не дают ROI** (Сколково AI Lab / TAdviser) [E]
- **75% компаний с бюджетом на AI не имеют стратегии внедрения** (MTS Web Services, 700+ компаний) [E]
- Пример: компания купила ChatGPT Pro на 500 сотрудников → через месяц **23 используют** [E]
- **33% компаний** сталкиваются с активным сопротивлением сотрудников [E]
- Surf: "Выдать подписку на Cursor — недостаточно. Без методологии большинство продолжат работать по-старому" [E]

**Вывод:** Методология внедрения — не фича, а ОСНОВНОЙ VALUE PROP продукта. Формулировка "мы продаём не AI-инструмент, а гарантию adoption" подтверждена рынком.

Рынок AI в РФ: **90+ млрд ₽** корпоративных расходов (2025), рост 25-40%/год. **58% средних компаний** (250-999 чел.) уже применяют AI. Но **только 5.8% реально развернули** эффективно. Разрыв adoption-execution — ядро рыночной возможности. [E]

## Hypothesis Validation Results

### Hypothesis 1: Оптовая торговля / Дистрибьюторы — **CONFIRMED [E]**

**Статус:** Подтверждён. Primary сегмент.

**Что подтвердила research:**
- CRM-конкуренция НЕ блокер: BitrixGPT (бывший CoPilot) и amoAI **не умеют генерировать КП из прайс-листов, не ищут по внутренним документам, не пишут автономные ответы на письма** [E, обзоры CRM]. Ядро use case продукта — непокрытый gap.
- Ability to pay: IT-бюджеты 3-15M₽/год. Продукт 80-110K/мес = 6-15% среднего бюджета. Конкурент SalesAI стоит 39-56K₽ (10 менеджеров), imot.io 100-150K₽/мес → наш ценник в рынке [E]
- AI-ready: 39% компаний используют AI-агентов для автоматизации БП (Сбер Аналитика, ноябрь 2025) [E]

**Что ИЗМЕНИЛОСЬ:**
- Главный риск уточнён: не CRM-конкуренция (она слабая), а **third-party интеграции** (n8n + amoCRM/Bitrix24 + ChatGPT/Claude) могут частично реплицировать функционал дешевле. Но: нет on-prem, нет методологии, нет обучения.

**Confidence upgrade:** Была [E] → осталась **[E]** с усиленными доказательствами.

**What we still don't know:**
- Конверсия из демо в покупку при цене 80-110K → нужен outreach
- Как позиционировать against "мы наняли фрилансера, он настроил n8n"

### Hypothesis 2: Производство с офисным блоком — **CONFIRMED [E]**

**Статус:** Подтверждён. Primary сегмент.

**Что подтвердила research:**
- **Нулевая конкуренция:** Habr/LANIT (авг. 2025): "среди найденных мной инструментов нет ни одного отечественного" для AI-поиска по техдокументации [E]. АСКОН — никакого AI, Leo.AI — "агрессивный маркетинг, непригодно для проектирования" [E]
- On-prem подтверждён как обязательный: 60% компаний в РФ инвестируют в on-prem AI, до 70% проектов разворачиваются на закрытой инфраструктуре [E]. ЛОЦМАН получил сертификацию ФСТЭК (январь 2026) — планка безопасности высока [E]
- Ability to pay: IT-бюджеты 4.1% от выручки → 4.1-41M₽ для целевого сегмента. 21% средних компаний готовы выделить 8-10% бюджета на AI [E, НАФИ]
- Сигналы боли: **3,146 вакансий КОМПАС-3D** на HH.ru, в требованиях — управление документацией, ГОСТ, спецификации [E]

**Confidence upgrade:** Была [E] → осталась **[E]** с сильнейшими данными. Пустое конкурентное поле делает этот сегмент стратегически ценным.

**What we still don't know:**
- On-prem deployment: готов ли продукт технически? (уточнить у клиента)
- Реакция на цену в живом разговоре с техдиректором

### Hypothesis 3: IT-компании — **CONFIRMED [H-high → E]**

**Статус:** Build-vs-buy гипотеза опровергнута. Сегмент валиден.

**Что подтвердила research:**
- Build-vs-buy ослаблен: ChatGPT **функционально заблокирован** в РФ (нет оплаты, IP-блоки) [E]. Кастомная разработка AI-ассистента стоит **200K-1M₽ upfront** + maintenance [E]. ТеДо (бывш. PwC Russia) **купил AutoFAQ** вместо разработки [E]. Doctolib, построив in-house, был "overwhelmed with feature requests within days", пришлось утроить data science команду [E].
- Job exists: MTS Web Services — инженеры тратят до **20% времени** на поддержку документации [E]. Новые сотрудники — до **30% времени на поиск информации** [E]. Уход senior-разработчика → **двухнедельный knowledge audit** [E]. 43% компаний без структурированного онбординга [E].
- Ability to pay: 80-110K₽/мес = **0.5-0.7 FTE разработчика** по московским ставкам [E]. ChatGPT Team ~$1250/мес на 50 чел — НЕДОСТУПЕН в РФ [E]. GigaChat API 5-30K₽/мес — но без RAG, UI, интеграций, внедрения.
- Прямые конкуренты: Minervasoft (50-150K₽/мес, база знаний + AI copilot) и AutoFAQ Xplain (AI для IT-support, RAG) — существуют, но нет методологии.

**Confidence upgrade:** Была [H-high] → теперь **[E]** на основании конкретных фактов.

**Уточнение сегмента:** HighTime Media SaaS рейтинг — **94 продуктовых компании** с выручкой 50M₽+ → конкретный prospect pool [E].

**What we still don't know:**
- Позиционирование against Minervasoft (прямой конкурент, сходная цена)
- Реакция IT-ЛПР (CTO/VP Engineering) на "методологию внедрения" — нужна ли им она или воспринимается как overhead?

### Hypothesis 4: Частная медицина — **REFINED [H-mid → H-high], но с ценовым ограничением**

**Статус:** Job подтверждена. Но сегмент сужен: **только клиники с выручкой 300M₽+**.

**Что подтвердила research:**
- Job exists сильно: среднее **120 звонков/день**, **80% вопросов — рутина** (расписание, подготовка, цены) [E]. Администраторы "полдня на телефоне" [E]. V-AI Labs: 72% обращений без участия человека, рост записей в 1.6 раза [E]. **338+ вакансий администраторов** в Москве — сигнал текучки и нехватки [E].
- Регуляторика благоприятна: ФСТЭК **НЕ обязателен** для частных клиник (только гос/муниципальные ИС по приказу ФСТЭК №17) [E]. Для on-prem нужна только "оценка эффективности" по ФЗ-152 [E]. AI НЕ нужна сертификация как мед. изделие, если работает только с административными задачами [E].
- Нет AI в МИС: МЕДИАЛОГ, Инфоклиника, 1С:Медицина — **никакого LLM-level AI** для административных задач [E]. Рыночный gap подтверждён.

**ЦЕНОВОЕ ОГРАНИЧЕНИЕ:**
- Большинство клиник платят **2-28K₽/мес за МИС** → наши 80-110K = **3-40x текущих IT-расходов** [E]
- Только при выручке **300M₽+** продукт = 0.2-0.4% выручки и ≈ стоимость 1-2 администраторов [E]
- V-AI Labs продаёт AI-ассистент за **105,000₽ разово** с окупаемостью 2 мес — ценовой якорь [E]
- Целевой TAM: ~**200 клиник** с выручкой 200M₽+ (Vademecum TOP200, позиции 45-80 — sweet spot)

**Конкуренция:** Чат-боты (V-AI Labs, Chatme.ai, TWIN, Syntera) от **~950₽/мес** — узкие, но дешёвые. Продукт дифференцируется полным набором задач (не только запись).

**Confidence:** Была [H-mid] → теперь **[H-high]** при ограничении выручкой 300M₽+. Маленький TAM (~200 клиник) делает это нишевым сегментом, не primary.

**What we still don't know:**
- Реакция главврача/управляющего клиники на "AI для администрации"
- Готовность к on-prem — большинство клиник на облачных МИС

### Hypothesis 5: Госорганизации — **REFINED [H-mid], HARD BLOCKER: Реестр ПО**

**Статус:** Job и ability to pay подтверждены. Но **Реестр российского ПО — жёсткий блокер** без обходных путей.

**Что подтвердила research:**
- Закупочный путь exists: годовой контракт 960K-1.32M₽ вписывается в лимит электронной малой закупки **5M₽** (ч.12 ст.93 44-ФЗ) — без полного тендера [E]. По 223-ФЗ (ГУПы/МУПы) пороги обычно **500K-5M₽** [E]. Малые электронные закупки значительно быстрее.
- CorpGPT слабый: молодая платформа, no-code конструктор ботов — не полноценный AI-ассистент. Гос.внедрения не задокументированы [E]. Directum Ario — ECM/СЭД с AI-модулями для классификации документов, **другая категория** [E]. AI-adoption в госсекторе **~2%** → рынок формируется [E].
- Ability to pay: IT-бюджеты ГУПов оценочно **3-15M₽/год**. Нацпроект "Экономика данных" — **457 млрд ₽** до 2027 → давление на цифровизацию [E].

**HARD BLOCKER:**
- Постановление Правительства №1236 **запрещает закупку иностранного ПО** по 44-ФЗ [E]
- Без Реестра продукт **юридически приравнивается к иностранному ПО** [E]
- CorpGPT **уже в Реестре** (запись №23046) → исключение "нет отечественного аналога" не работает [E]
- Регистрация: 2-4 мес., **только 21% заявок проходят** [E]. Требования: ≥50% российского владения, исходный код в РФ, совместимость с Astra Linux/ALT Linux, без иностранных СУБД [E]

**Вывод:** Сегмент **заморожен до входа в Реестр ПО**. Начинать регистрацию НЕМЕДЛЕННО — это блокирующая зависимость. Тем временем можно работать с ГУПами/МУПами по **223-ФЗ** (более гибкие закупки) как первая волна.

**Confidence:** Осталась [H-mid]. Потенциал высокий, но реализуем только после снятия блокера. FL-совместимость частично подтверждена (малые закупки без тендера).

**What we still don't know:**
- Может ли продукт пройти регистрацию в Реестре? (зависимости от иностранных СУБД? LLM?)
- Сроки реальной регистрации — 2 или 12 месяцев?
- ГУПы по 223-ФЗ — готовы ли покупать БЕЗ Реестра?

### Hypothesis 6: Логистика / Транспорт — **KILLED [E-blocker: ability to pay]**

**Статус:** Убит на основании evidence-backed blocker.

**Почему убит (3 фактора):**

1. **Ability to pay — блокер [E]:** При рентабельности 4% компания с выручкой 100M₽ зарабатывает **4M₽ чистой прибыли**. Продукт за 960K-1.32M₽/год = **24-33% чистой прибыли** — нереально для рационального покупателя. Даже при выручке 500M₽+ (прибыль ~20M₽) продукт = 5-7% прибыли — тяжёлая строка расходов. [E, отраслевая статистика рентабельности]

2. **ЭДО-триггер нерелевантен [E]:** Обязательная Э-ТТН с сентября 2026 обслуживается специализированными операторами ЭДО — **1С-ЭПД, Контур, Астрал, СБИС**. Это юридически регулируемые, структурированные документы с КЭП. **Нулевое пересечение** с функциональностью AI-ассистента. Триггер ложный. [E, анализ ЭДО-решений]

3. **AI в логистике ≠ офисная рутина [E]:** Ассоциация "Цифровой транспорт и логистика" — 45% компаний планируют AI, но use cases: оптимизация маршрутов, прогнозирование спроса, складская робототехника — **не офисная обработка документов и клиентская переписка**. Конкретных примеров AI для офисной работы в средних логистических компаниях РФ не найдено. [E]

**Примечание:** Клиент в REDO-заказе сам указал: "тут у них может не быть денег на цифровую трансформацию" [E, founder input]. Исследование подтвердило.

### Hypothesis 7: Сервисные компании (юр./бух. аутсорс) — **REFINED: подсегмент "крупные юр. фирмы"**

**Статус:** Массовый сегмент убит на ценовом блокере. Но **крупные юр. фирмы (топ-20 Право-300, выручка 500M₽+)** — жизнеспособный нишевый сегмент.

**Что подтвердила research:**
- Job exists сильнейше: **88% юристов в РФ уже используют AI** (Avito + Право.ru, конец 2025) — максимальный adoption [E]. 500+ договоров/год = 1000-2000 чел.-часов [E]. On-prem критичен из-за адвокатской тайны [E].
- Нет конкурента с полным набором: КонсультантПлюс AI — только по базе К+, не внутренние доки [E]. Гарант ИСКРА — только публичная база [E]. Doczilla Pro 2900₽/чел/мес — нет анализа встреч, email, внутреннего поиска [E]. **Ни один продукт не объединяет: внутренние документы + интернет + email + встречи + on-prem** [E].
- Ability to pay (крупные): Топ-20 по выручке = 500M₽-5B₽+. IT-бюджет 3-5% → **15-250M₽**. Продукт ~1M₽/год = 0.4-7% IT-бюджета — легко [E].

**ОГРАНИЧЕНИЕ:**
- TAM: ~**50-80 юр. фирм** и 15-30 крупных бух. аутсорсеров с 50+ чел. и выручкой 200M₽+ [E]
- При конверсии 20% → потенциальный ARR **14-24M₽/год** — недостаточно как primary, но хорош как beachhead для юр. сектора [E]

**Confidence:** Была [H-mid] → теперь **[H-high]** для крупных фирм. TAM мал, но pain и willingness to pay высоки.

**What we still don't know:**
- Реакция управляющего партнёра на "методологию внедрения AI" — не воспримут ли как снисхождение?
- Расширение: после юр. фирм → корпоративные юр. отделы (гораздо больший рынок)

### Hypothesis 8: Тренинговые компании — **KILLED [E-blocker: размер рынка + ценовой fit]**

**Статус:** Убит на основании evidence-backed blockers.

**Почему убит:**

1. **Размер рынка [E]:** Тренинговых компаний с выручкой >50M₽ — **менее 500** в РФ. Большинство — микро-предприятия и соло-тренеры. [E, рыночные данные]

2. **Ценовой fit [E]:** 80-110K₽/мес для 10-15 чел. = **6-8K₽/чел/мес** — в 50x дороже iSpring LMS (131₽/чел/мес). Корп. университеты с бюджетами 557M₽ — enterprise, не наш таргет. [E, pricing data]

3. Пилоты у клиента в "обучающих компаниях" [E, founder input] возможно относятся к корпоративным учебным центрам при крупных компаниях — это enterprise, не средний бизнес.

### Hypothesis 9: Девелоперы / Строительство — **REFINED: emerging, но НЕ сейчас**

**Статус:** Отложен. Потенциал подтверждён, но готовность рынка низкая.

**Что подтвердила research:**
- Job exists: ГК ФСК (AIVOR) — снижение времени подготовки документов на **45%**, рост продуктивности **50%** [E]. ГК КОРТРОС — "интеллектуальная корпоративная база знаний" [E]. 70% девелоперов заинтересованы в AI-моделях [E].
- НО: строительство — **одна из наименее цифровизованных отраслей РФ** [E]. IT-расходы <1% выручки (vs 5% в ретейле) [E]. Цифровая зрелость низкая.
- Конкурент: НейроШтат — **50 виртуальных AI-сотрудников** для строительных компаний, on-prem опция → прямой конкурент [E].
- Нужна отраслевая специфика (СНиП, ГОСТ, методологии Минстроя) — generic продукт может не покрыть.

**Вывод:** Мониторить. Готовить отраслевую адаптацию. Целить в компании с выручкой 1B₽+. Не вкладывать ресурсы FL сейчас.

## Decisions Made (FL perspective)

1. **IT-компании → Tier 1.** Клиент подтвердил работу с ними, research подтвердил job и ability to pay, CTO/VP Engineering хорошо доступны через LinkedIn/HH.ru. Без вопросов.

2. **Госорганизации → заморожены.** Реестр ПО — блокер, FL не ждёт. Рекомендация клиенту: начать регистрацию в Реестре ПО параллельно с outreach по другим сегментам. Когда/если Реестр пройден → разморозить.

3. **Частная медицина → Tier 2, после primary.** TAM ~200 хватает для FL-теста (нужно 50), но не приоритет пока в Торговле 5000-15000, Производстве 6000-10000 и IT 94+ компании. Тестируем если будет свободный sprint.

4. **Производство vs Торговля → тестируем параллельно.** FL может запустить два кампании одновременно. "Beachhead" определяется результатами outreach, а не на бумаге. Нулевая конкуренция в Производстве — сильный аргумент, но FL тестирует оба и данные решают.

5. **On-prem → не блокирует outreach.** FL тестирует спрос. Если проспекты спросят про on-prem — это сигнал для клиента, не стоп для кампании. Флагнуть в отчёте.

## Рекомендации клиенту (вне scope FL)

- Начать регистрацию в Реестре российского ПО (2-4 мес, 21% approval). Блокирует весь госсектор.
- Подтвердить техническую готовность on-prem deployment. Критично для Производства и IT.
