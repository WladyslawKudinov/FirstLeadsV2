# Monetization Agent — Changelog

## v2 — 2026-02-20

**Pricing Philosophy + Product Stage Assessment** — два системных фикса:

### Fix 1: Step 0.5 — Pricing Philosophy (BEFORE mechanical model-building)

Новый обязательный шаг с Decision Card по стратегическому подходу:

**A. Product Stage Assessment**
Сканировать Value Map на confidence ratings:
- **Pre-traction** (0 proof points) → MUST include pilot pricing + risk-reversal
- **Early traction** (1-3 proof points) → should include case-study pricing + introductory rates
- **Proven** (3+ proof points) → can use standard tiered pricing

Стадия продукта теперь отображается в Резюме.

**B. Pricing Strategy Decision Card**
3-4 радикально разных подхода (НЕ "3 тарифа vs 4 тарифа"):
- Sell transformation (project + ongoing)
- Sell subscription (standard SaaS)
- Sell outcomes (pay-per-result)
- Freemium → upsell (PLG)
- Land-and-expand
- Pilot → convert

Каждый вариант включает: core logic, как усиливает позиционирование, buyer behavior, GTM mechanics, risks.

**C. Stage-Appropriate Mechanisms**
Pre-traction продукт БЕЗ pilot pricing и risk-reversal = критическая ошибка. Механизмы теперь обязательные, не опциональные.

### Fix 2: Rules 12-14

- **Rule 12: Think before you build** — не прыгать сразу к "3 тарифа" без Step 0.5
- **Rule 13: Product stage dictates pricing** — pre-traction CANNOT use same strategy as proven
- **Rule 14: Don't parrot positioning doc** — думать, а не копировать

### Output Format Update

- Добавлена секция "1. Стратегия ценообразования (Decision Card)" — теперь первая в main body
- Резюме теперь показывает "Стадия продукта" с reasoning
- "Обязательные механизмы для данной стадии" в каждом документе
